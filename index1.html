Інформація про системи контролю версій та Git

Система керування версіями - це система, що записує зміни у файл або набір файлів протягом деякого часу таким чином, що можна повернутися до певної версії пізніше. Це може бути як програмний код так і будь-які інші файли. Існують три основні типи систем керування версіями: локальні, з централізованим сховищем та розподіленим (деценралізованим).

Git – це система контролю версій, яка дозволяє відстежувати зміни у коді та співпрацювати з іншими розробниками. Вона була створена Лінусом Торвальдсом у 2005 році для управління розробкою ядра Linux. Git відрізняється від інших систем контролю версій тим, що кожен розробник має локальну копію всього репозиторію, а не лише останньої версії коду. Це дозволяє працювати незалежно та робити зміни без доступу до центрального сервера. Git також надає потужні інструменти для роботи з гілками, що дозволяє розробляти декілька функцій одночасно та об'єднувати їх в один код.

Основні можливості та переваги використання Git

ОСНОВНІ МОЖЛИВОСТІ

КОНТРОЛЬ – дозволяє спостерігати за змінами в коді та здійснювати виправлення без стресу.

СПІВПРАЦЯ – можна працювати в команді над великими проєктами без конфліктів.

БЕЗПЕКА – дає можливість відновлювати видалені файли та повертати зміни за допомогою історії комітів.

ГНУЧКІСТЬ – можна створювати гілки для різних функцій та експериментувати без ризику для основного коду.

ПЕРЕВАГИ

ШВИДКІСТЬ – Git оптимізований для швидкої роботи з репозиторіями.

ПРОСТОТА ВИКОРИСТАННЯ – має інтуїтивно зрозумілий інтерфейс командного рядка.

ПІДТРИМКА НЕЛІНІЙНОЇ РОЗРОБКИ – Git підтримує створення і злиття гілок, що дозволяє розробникам працювати над різними функціями або виправленнями паралельно, без переривання основної лінії розробки.

Базові команди роботи в системі Git

git init 			ініціалізує новий Git репозиторій в поточній директорії

git status 			виводить стан робочого дерева та індексу

git log 			виводить історію комітів

git branch 			виводить список гілок у репозиторії

git add < file > 		додає вказаний файл до індексу

git switch < branch > 		перемикається на іншу існуючу гілку

git merge < branch > 		зливає зміни з вказаної гілки в поточну гілку

git checkout 			переключається на вказаний коміт

git reset 			скасовує всі зміни до вказаного коміта

git show 			показує зміни між комітом і поточним станом файлу
